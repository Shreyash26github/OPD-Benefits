Policy Seeding Mechanism
Product Configuration SOP
System: Salesforce Vlocity  ·  Purpose: Standard process for configuring products in Vlocity
Objective
To provide a standardized and repeatable process for configuring products in Vlocity, including benefits, attributes, pricing, entitlements, and product relationships.
Scope — This SOP Covers
  • Product creation and cloning
  • Benefit and attribute configuration
  • Network / Query mapping (Product Relationship)
  • Entitlement management
  • Product source & channel
  • Pricing setup using BRE
  • Validation, Testing & Deployment
Output
✓ Fully configured product ✓ Accurate pricing & entitlements ✓ Network mappings complete ✓ Product ready for business use
1
Product Creation
  • Navigate to the Product object in Vlocity
  • Create a new product record
  • Enter: Product Name · Product Code · Product Tag · Product Category and related fields
  • Save the product record
1.2  Product Cloning (If Applicable)
  • Use Deep Clone (Batch) functionality in Salesforce to clone an existing product
  • Ensure all related records are cloned correctly
  • Update the cloned product with new Product Name, Product Code, Product Tag and other required details
Salesforce Screenshot — Product Coverages
Salesforce Vlocity
Product view showing benefit coverages: Doctor Consultation, Lab and Radiology, Dental Care, Pharmacy Wallet listed under Star Health OPD Plan
2
Benefit & Attribute Configuration
2.1 Benefit (Coverage) Configuration
  • Navigate to the Add Benefits section under the product
  • Add benefits as per the product design (e.g. Benefit Name: Doctor Consultation)
2.1.1 Claimable Attributes
Update value of respective benefits
e.g. Doctor Consultation wallet ₹10,000, 2 PHC vouchers, etc.
2.2.2 Non-Claimable Attributes
Update utilization values:
Cashless – True, Reimbursement – True, FUPs if any
Salesforce Screenshot — Attribute Configuration
Salesforce Vlocity
Attribute table showing claimable and non-claimable attributes for Doctor Consultation (family wallet) — Family OPD Wallet ₹50,000, Access Level, FUP settings
3
Product Relationship — Network & Query Mapping
  • Use the Product Relationship object to define network mappings
  • Map the product benefit to: Hospitals network · Doctors specialization/fees allowed (query) · Labs network
  • This configuration enables network-based listing in the application
Salesforce Screenshot — Product Relationship (Network Mapping)
Salesforce Vlocity
Product Relationship record showing Network type mapping for Lab and Radiology — Health Prime Network, Index Value: Labs
4
Entitlement Configuration
  • Navigate to the custom object Entitlement
  • Create or update records for: Inclusions · Exclusions · Documents · Procedures
  • Map each entitlement record to: Product · Benefit · Attribute
Salesforce Screenshot — Entitlement Documents

5
Product Source & Channel Configuration
  • Configure Source and Channel fields on the product
  • Used for: Business source identification · Partner identification
Example: Source – eBH  ·  Channel – GMC
Salesforce Screenshot — Product Source & Channel

6
Pricing Configuration — Matrix & BRE
  • Enable the Pricing Matrix for the product
  • Use Business Rule Engine (BRE) for pricing calculations and validations
BRE Setup Steps
1Store pricing logic and values in a CSV file using the Product Code
2Upload the CSV file into BRE
3Configure pricing rules and procedures in BRE
4Execute the rules to calculate pricing
5Verify the final pricing output
7

✓  Test network listings (hospitals, doctors, labs)
